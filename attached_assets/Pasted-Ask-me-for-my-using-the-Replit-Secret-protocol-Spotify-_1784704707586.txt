28	Alarm widget creates new AudioContext on every tick — playAlarmSound inside setInterval creates a fresh context each time; mobile browsers cap simultaneous AudioContexts and the alarm sound dies mid-ring.
29	Orb animation is expensive on low-end Android — Orb runs 8–10 particle loops plus 4–5 core orb loops; causes frame drops on mid-range devices.
30	Weather forecast scrollbar is too thin to grab — index.css sets scrollbar width to 4px; on touch screens there is no visible handle.

41	AlarmWidget can be up to 5 seconds late — checks only every 5 seconds; background throttling on mobile makes it worse.

43	1 GB file upload can crash mobile tabs — FileReader.readAsDataURL() base64-encodes the whole file in memory, then stores it in React state; large files crash low-RAM devices.
44	playWakeSound() leaks AudioContext on every wake word — each "hey Jarvis" detection creates a new AudioContext that is never closed, eventually exhausting browser audio limits.
45	No theme-color meta tag — Android Chrome address bar stays default colour instead of matching the app theme.
