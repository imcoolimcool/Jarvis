Build a voice AI assistant prototype called "Jarvis".

I already have all required API keys ready:
- Whisper-large-v3 API for speech-to-text
- LLM API for reasoning/conversation
- Text-to-speech API for voice generation

Your job is to build the full application that connects these APIs.

Core flow:

1. User presses a microphone button.
2. The app records the user's voice.
3. Audio is sent to Whisper-large-v3.
4. Whisper returns the transcript.
5. The transcript is sent to the LLM with a special system prompt that tells it:
   - It is Jarvis, a personal voice assistant.
   - It is currently in voice mode.
   - Responses must be short, natural, and conversational because they will be spoken aloud.
   - Avoid markdown, lists, and long explanations unless requested.
   - Sound like a helpful human assistant.
6. The LLM response is sent to the TTS API.
7. The generated audio is played back to the user.

The assistant should feel like a real conversational voice AI, not a chatbot.

Features for version 1:
- Clean modern interface.
- Large microphone button.
- Animated listening state.
- Show the transcript of what the user said.
- Show the assistant response text.
- Play the generated voice response automatically.
- Conversation history during the session.
- Ability to interrupt/stop the assistant speaking.
- Loading states for listening, thinking, and speaking.

Technical requirements:
- Use a proper frontend/backend separation.
- Never expose API keys in frontend code.
- Store API keys securely using environment variables.
- Create backend routes for:
  - audio upload
  - Whisper transcription
  - LLM request
  - TTS generation
- Handle errors gracefully.
- Make the code modular so future features can be added.

Architecture:

Frontend:
- Voice assistant UI
- Microphone recording
- Audio playback
- Conversation display

Backend:
- API integrations
- Prompt management
- Audio processing
- Security

Create a configuration file where I can easily change:
- LLM model
- TTS voice
- Jarvis personality prompt

The first version does NOT need:
- Smart home controls
- Calendar
- Memory database
- Wake word detection
- Phone integrations

Focus only on creating the best possible "talk → think → speak" Jarvis experience.

Make the UI polished and futuristic, inspired by a high-end AI assistant interface.
